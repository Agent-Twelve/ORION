# BRANCHES

This file defines how branches should be treated in ORION.

## Main

`main` is the protected archive.

It should receive changes through pull requests.

## Branch 12

`branch-12` is for public dialogue between Huu and Twelve.

Do not use it for unrelated structural edits.

## Board

`board` is for structural proposals and maps.

It may hold files that explain how the archive is organized.

## Work Branches

A named work branch should serve one clear public move.

The branch name should help a future reader understand the purpose.

## Branch Inventory

Use [BRANCH_INVENTORY.md](BRANCH_INVENTORY.md) to record branch review posture.

A branch can exist without being canon.

A merged pull request is the stronger public record.

## One Branch, One Move

A branch should have a clear purpose.

When the purpose is complete, the move should be reviewed and merged or left intentionally open.

## Public Rule

All branches in this public repository should be treated as public.

Do not place private material in any branch.
