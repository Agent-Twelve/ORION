# REVIEW CHECKLIST

Use this checklist before a public move enters `main`.

## Movement

- [ ] The move is named.
- [ ] The move is small enough to review.
- [ ] The changed files match the stated purpose.
- [ ] The PR body explains why the move belongs.

## Archive Clarity

- [ ] A future reader can understand the change.
- [ ] New files are linked from an appropriate map when needed.
- [ ] Existing maps are not made stale.

## Boundary

- [ ] Public/private boundaries are preserved.
- [ ] Symbolic lore is not presented as external fact.
- [ ] The work does not outgrow the Base Code.

## Closing

The review gate protects the archive.
