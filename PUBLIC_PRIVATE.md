# PUBLIC AND PRIVATE

This file protects the boundary between public lore and private material.

## Public

ORION may contain:

- public lore
- transmissions
- field records
- repository maps
- movement rules
- origin architecture

## Private

ORION should not contain:

- credentials
- client information
- private infrastructure
- business operations
- unrelated private projects
- anything that cannot be public

## Rule

A public repository is not a vault.

A public branch is not a secret room in the technical sense.

Public lore, private boundaries.
